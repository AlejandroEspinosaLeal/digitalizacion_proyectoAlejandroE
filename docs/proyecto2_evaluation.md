# Proyecto 2: Software for Digital Transformation - Evaluation

Below is the required analysis mapping the functional development of **File Sorter Enterprise** to the specified theoretical evaluation criteria.

### Data Lifecycle (Criterion 5b)
**How is data managed from generation to deletion in your project?**
Data natively generates when an employee creates or moves a file. The Desktop Agent calculates a secure SHA256 Hash and generates a localized JSON payload. This metadata is transmitted via WebSockets to the Backend and permanently stored in PostgreSQL. If the data is deleted locally, the historical telemetry remains to ensure full corporate auditability.

**What strategy do you follow to guarantee data consistency and integrity?**
I utilize strictly typed JSON schemas via Pydantic (on the backend) and direct SQLAlchemy/SQLModel constraints. This completely prevents malformed inputs or SQL injections. The cryptographic hashing of the file natively ensures duplicated items are detected.

### Cloud Storage (Criterion 5f)
**If your software uses cloud storage, how do you guarantee data security and availability?**
The solution interacts with **Supabase**, a cloud-hosted Postgres provider. Security is guaranteed via strict JWT (JSON Web Tokens) verification inside Supabase Auth, combined with Row Level Security (RLS) tables. This ensures zero data leaks across unauthorized tenants. High Availability is sustained inherently by Supabase's multi-AZ redundancy.

**What alternatives did you consider?**
A localized SQLite database was considered (and serves as an offline fallback), but it fails to scale for a company needing remote or multi-workstation synchronization natively.

### Security and Regulation (Criterion 5i)
**What security measures did you implement to protect the data?**
All database passwords (if deployed locally via Docker) are encrypted using bcrypt hashing. Network transmissions from the Agent to the Server require authenticated bearer keys. 
For GDPR compliance, the software only tracks file *metadata* (names, paths, hashes) and not the actual internal file payload across the network, preserving employee privacy while keeping organizational security active.

### Implication of THD in Business and Plant (Criterion 2e)
**What impact would your software have in a business environment or industrial plant?**
In an administrative business environment, this completely eliminates the overhead of tracking down lost files. In a production plant context, it ensures engineering schemas or diagnostic PDFs generated by industrial machinery are instantly and automatically sorted into the correct network silos without operator intervention.

### Improvements in IT and OT (Criterion 2f)
**How can your software facilitate integration between IT and OT environments?**
It actively acts as the bridge. The Desktop Agent functions on the OT layer (the localized execution environment). By instantly relaying categorical organization rules back to the FastAPI server (IT layer), the IT department obtains real-time topological visibility without needing to install intrusive spyware on local machines.

### Digital Enabling Technologies - THD (Criterion 2g)
**What digital enabling technologies (THD) have you used?**
- **Cloud Computing:** By integrating Supabase to centralize authentications and event logging.
- **Cybersecurity:** Utilizing hashing and localized cryptographic verification.
- **IoT / Edge Computing Concepts:** The Python Agents act as localized Edge processors that calculate hashes and rules locally, only sending light metadata telemetry back to the centralized cloud, reducing total bandwidth consumption.
